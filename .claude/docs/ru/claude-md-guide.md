# Как написать эффективный CLAUDE.md

> Полный гайд на английском в [claude-md-guide.md](/docs/claude-md-guide). Здесь приводится сокращённая версия.

## Золотые правила

1. **Краткость** — ≤200 строк, иначе CLAUDE.md перестанут читать
2. **Конкретные ограничения** — "max 400 lines per file" вместо "файлы должны быть короткими"
3. **Команды для копи-паста** — не пересказывай CLI-help, дай конкретные строки
4. **Hard vs Warning vs Instruction** — пометь каждое правило по уровню enforcement
5. **Без сценариев "если"** — пиши имеющиеся правила, не гипотетические

## Структура

```
# Project Name (1 line)

# Стек / Точка входа / БД / Тесты (≤5 lines)

## Принципы (≤10 строк)

## Ограничения (жёсткие, hard-enforced)

## Архитектура (краткое описание + где смотреть детали)

## Команды (≤15 типичных команд)

## Current State (DYNAMIC block, авто-обновляется)
```
