# Инструмент: draft

## Описание

Генерирует черновик ответа на входящее сообщение через локальную LLM (Phi-3-mini GGUF).

Автоматически подставляет контекст: текущие товары и график работы. Используется для тем "Цены" и "График". Работает полностью локально, ~3-5 секунд на сообщение.

### Команда

```bash
ai-mini-box draft [OPTIONS] TEXT
```

### Аргументы

| Аргумент | Тип | Описание |
|----------|-----|----------|
| `TEXT` | STRING | Исходное сообщение клиента |

### Опции

| Опция | Описание |
|-------|----------|
| `--contact TEXT` | Имя контакта (для персонализации) |
| `--topic [Цены\|Заказ\|Жалоба\|График\|Другое]` | Тема сообщения (если известна) |
| `--model PATH` | Путь к GGUF-файлу модели |
| `--ctx N` | Контекст LLM в токенах (default: 4096) |
| `--threads N` | Количество потоков CPU (default: 4) |
| `--no-products` | Не подставлять товары в контекст |
| `--no-schedule` | Не подставлять график работы |
| `--json` | Вывод в JSON |
| `--abort` | Прервать генерацию (для долгих запросов) |

### Примеры

```bash
ai-mini-box draft "Сколько стоит доставка?"
# → Черновик: "Здравствуйте! Стоимость доставки составляет..."

ai-mini-box draft --contact "Иван" "Есть ли у вас товар X?"
# → Черновик: "Иван, здравствуйте! Да, товар X в наличии..."

ai-mini-box draft --json "Можно прийти в субботу?"
# → {"draft": "...", "model": "Phi-3-mini", "tokens": 128, "time_ms": 3400}
```

## Промпт для вайбкодинга

```markdown
Создай CLI-инструмент `draft` для генерации черновиков ответов через локальную LLM.

### Требования:
1. Используй Typer для CLI
2. Используй существующий `LlamaCppProvider` из `infrastructure/llm/llama_cpp_provider.py`:
   - Конструктор: `LlamaCppProvider(model_path, n_ctx, n_threads)`
   - Метод `generate_draft(text, products, schedule) -> str`
   - Метод `abort()` — прерывает текущую генерацию
3. Lazy-load: модель загружается при первом вызове
4. Используй существующий `DraftService` из `core/services/draft_service.py`:
   - Принимает `LlamaCppProvider`, `ProductRepo`, `AppConfig`
   - Метод `generate(text, contact_name, topic) -> str`
5. Флаг `--json` для структурированного вывода (draft, model, tokens, time_ms)
6. Таймаут генерации 30 секунд
7. Контекст товаров и графика подтягивается из БД (ProductRepo, AppConfig)
8. Если модель не загружена — понятная ошибка
```
