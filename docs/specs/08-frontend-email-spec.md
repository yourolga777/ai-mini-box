## 3. Состояния

| Состояние | Что показываем |
|---|---|
| Loading | Skeleton формы |
| Not configured | Пустая форма, все поля пустые, статус «○ Не настроено» |
| Configured | Форма заполнена, статус из `/api/email/status` |
| Testing | Кнопка «Проверка...» disabled, спиннер |
| Test success | Toast «Подключение успешно» |
| Test error | Toast с ошибкой |
| Saving | Кнопка «Сохранение...» disabled |
| Save error | Toast с ошибкой |

## 4. API-вызовы

- `GET /api/email/status` — при монтировании + auto-poll 30s
  - Ответ: `{ configured, connected, last_poll_at, last_error, messages_fetched_today }`
- `POST /api/email/test` — кнопка «Проверить подключение»
  - Тело: `{ imap_host, imap_port, imap_ssl, smtp_host, smtp_port, smtp_ssl, email_address, email_password }` (текущие значения формы)
  - Ответ: `{ success, imap: bool, smtp: bool, message }`
- Сохранение — `POST /api/plugins/email/config` (через PluginManager, стандартный эндпоинт конфига плагина)

## 5. Валидация

- IMAP/SMTP host — не пустой
- Port — число от 1 до 65535
- Email — базовая проверка формата
- Password — не пустой (но скрыт, показывать только ●●●)
- `max_per_cycle` — число ≥ 1
- `poll_interval_seconds` — число ≥ 5

## 6. Критерии приёмки

- Форма отображается с текущими настройками (из status)
- Можно изменить поля и сохранить
- Кнопка «Проверить подключение» отправляет test-запрос
- Статус обновляется в реальном времени (auto-poll)
- Валидация полей перед отправкой
- Состояния загрузки/ошибки обработаны
- Пароль скрыт (поле type=password)
