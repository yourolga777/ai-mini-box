## 1. Поиск, дата и сортировка на странице сообщений

**Файлы:** `packages/web/frontend/src/pages/Messages.tsx`, `src/components/Table.tsx`

**Описание:**

**Критерии приёмки:**

## 6. Чат-диалог на странице контакта (P1-2)

**Файлы:** `packages/web/frontend/src/pages/ContactDetail.tsx`

**Описание:**
Переработать отображение сообщений на странице контакта `/contacts/:id` из списка-карточек в формат диалога (как в мессенджере).

**Детали реализации:**

**1. Чат-пузырьки вместо списка:**
- Входящие сообщения (от клиента) — слева, серый фон, скруглённые углы
- Исходящие (ответы менеджера, поле `draft_response`) — справа, синий фон
- Текст сообщения полностью, без обрезания (`slice(0, 80)` убрать)
- Дата/время (`formatDate(received_at)`) под каждым пузырьком мелким шрифтом
- Topic badge (цветной, `TOPIC_COLORS`) на входящих сообщениях
- Если `sent_response=true` — галочка ✓ на исходящем пузырьке
- Автоскролл к последнему сообщению после загрузки (`useEffect` + `scrollIntoView`)

**2. Поле ввода ответа:**
- `textarea` (многострочное) + кнопка «Отправить» внизу списка сообщений
- При отправке — `POST /api/messages/{id}/reply` с телом `{ "text": "..." }` для последнего входящего сообщения (или для того, на которое ещё не отвечали)
- После успешного ответа — новый пузырёк появляется в диалоге (добавить в локальный стейт, без перезагрузки)
- Кнопка заблокирована (`disabled`) пока идёт отправка
- При ошибке — текст не теряется, показать уведомление (alert или красный текст над полем)
- Если сообщений нет — поле ввода скрыто или отключено

**API (уже существует, не менять):**
- `GET /api/messages?contact_id=:id&limit=100` — получить все сообщения контакта
- `POST /api/messages/{id}/reply` с `{ "text": "..." }` — отправить ответ

**Критерии приёмки:**
- Входящие пузырьки — слева, исходящие — справа (визуально как Telegram/WhatsApp чат)
- Текст сообщения виден целиком, не обрезан
- После отправки ответа он сразу появляется в диалоге (без F5)
- Скролл автоматически уходит вниз к последнему сообщению
- При пустом диалоге (нет сообщений) — надпись «Нет сообщений» и поле ввода скрыто

## 7. UI плагинов: установленные vs доступные + обновления (P2-1)

**Файлы:** `packages/web/frontend/src/pages/Plugins.tsx`, `src/components/InstallModal.tsx`

**Описание:**
Разделить страницу плагинов на два таба: «Установленные» и «Доступные». Добавить кнопку обновления для установленных плагинов, если есть новая версия.

**Детали реализации:**

**Два таба:**
- **Установленные** — загружается из `GET /api/plugins`. Карточка: имя, описание, версия, статус (работает/остановлен/не запущен), цветной индикатор. Кнопки:
  - «Запустить» / «Остановить» (если плагин с демоном)
  - **«Обновить»** (если `has_update=true`) — вызывает `POST /api/plugins/{name}/update`, показывает лог в модалке
  - **«Удалить»** — с подтверждением (как сейчас)
- **Доступные** — загружается из `GET /api/plugins/catalog`. Карточка: имя, описание, версия. Кнопка **«Установить»** (если `installed=false`, ведёт в InstallModal).

**Существующие страницы:** `PluginDetail.tsx` не трогаем.

**Критерии приёмки:**
- Табы переключаются без перезагрузки
- После установки/обновления — React Query инвалидирует кэш, список обновляется
- Если каталог пустой — «Нет доступных плагинов»
- Если установленных нет — «Нет установленных плагинов»
- Кнопка «Обновить» отображается только если `has_update=true`
- Во время обновления кнопка заблокирована, спиннер
- InstallModal остаётся без изменений

## 8. Архитектурные решения

| Решение | Выбор |
|---|---|
| Маршрутизация | React Router v6 |
| HTTP-клиент | fetch (обёртка в `api/client.ts`) |
| Стили | Tailwind CSS 4 |
| Структура страниц | `src/pages/<Name>.tsx`, компоненты в `src/components/<Name>.tsx` |
| Сортировка | server-side (`sort_by`, `sort_order` query params) |
| Поиск | server-side (`search` query param, debounced 300ms) |

| Что нельзя делать |
|---|
| Менять API-роуты на стороне фронта (они серверные) |
| Добавлять новые зависимости (React Router уже есть) |
| Писать стили в CSS — только Tailwind utility classes |
| Хранить состояние сортировки/поиска в URL (query params) — обязательно через URL |

## P2-2: Order UI в MessageDetail и ContactDetail

**Файлы:**
- `packages/web/frontend/src/pages/MessageDetail.tsx`
- `packages/web/frontend/src/pages/ContactDetail.tsx`

### 1. MessageDetail — блок заказа

Добавить блок «Заказ» между ContactLink и Reply:

```tsx
{/* Блок заказа */}
{linkedOrder ? (
  <div className="...">
    <h3>Заказ #{linkedOrder.id}</h3>
    <p>Статус: {statusLabel(linkedOrder.status)}</p>
    <p>Сумма: {formatPrice(linkedOrder.total_kopecks)}</p>
    <a href={`/orders/${linkedOrder.id}`}>Открыть заказ →</a>
  </div>
) : (
  <button onClick={handleCreateOrder}>
    Создать заказ
  </button>
)}
```

**Состояния:**
- `loading` — загружаем `GET /api/messages/{id}/order`
- `order exists` — показываем карточку заказа (ID, статус, сумма, ссылка)
- `no order` — кнопка «Создать заказ»
- `creating` — модальное окно: поле total_kopecks, поле notes, кнопки «Создать» / «Отмена»
- `created` — скрываем модалку, показываем карточку заказа
- `error` — toast об ошибке

**Модалка создания заказа:**
- Поле «Сумма (коп.)» — number input, необязательное
- Поле «Примечание» — textarea, предзаполнено текстом сообщения
- Кнопка «Создать» → `POST /api/messages/{id}/create-order`
- Кнопка «Отмена» → закрыть модалку
- Если сообщение не привязано к контакту — показать сообщение: «Сначала привяжите сообщение к контакту»

### 2. ContactDetail — блок заказов

Добавить секцию «Заказы» (после сообщений или рядом):

```tsx
<section>
  <h2>Заказы</h2>
  {orders.length === 0 ? (
    <p>Нет заказов</p>
  ) : (
    <table>
      <thead>
        <tr>
          <th>ID</th>
          <th>Статус</th>
          <th>Сумма</th>
          <th>Дата</th>
        </tr>
      </thead>
      <tbody>
        {orders.map(order => (
          <tr key={order.id}>
            <td><Link to={`/orders/${order.id}`}>#{order.id}</Link></td>
            <td>{statusLabel(order.status)}</td>
            <td>{formatPrice(order.total_kopecks)}</td>
            <td>{formatDate(order.created_at)}</td>
          </tr>
        ))}
      </tbody>
    </table>
  )}
</section>
```

**Загрузка:** `GET /api/orders?contact_id={contactId}`

### 3. Форматирование

- `formatPrice(kopecks: number)` — переводит копейки в рубли: `1 500.00 ₽`
- `statusLabel(status: string)` — читаемые статусы: `new` → «Новый», `processing` → «В обработке», `completed` → «Выполнен», `cancelled` → «Отменён»

### Критерии приёмки:
- В MessageDetail отображается заказ (если есть) или кнопка создания
- Модалка создания заказа открывается, отправляет запрос, закрывается при успехе
- После создания заказа — интерфейс переключается на карточку заказа
- В ContactDetail отображается таблица заказов контакта
- Сортировка заказов — по `created_at DESC`
- Для контакта без заказов — сообщение «Нет заказов»
