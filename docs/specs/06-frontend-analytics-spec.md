### Виджет KPI cards
4 карточки: новые сообщения, новые контакты, новые заказы, выручка сегодня.
Данные: `GET /api/analytics/summary`.

### Виджет «Динамика»
Line chart с тремя линиями на одном графике:
- Сообщения по дням (`/api/analytics/messages`)
- Заказы по дням (`/api/analytics/orders`)
- Выручка по дням (`/api/analytics/revenue`)
- Ось X: даты, ось Y: количество/сумма

### Виджет «Каналы»
Doughnut chart по `GET /api/analytics/channels`.

### Виджет «Топ контактов»
Список: `GET /api/analytics/top-contacts?limit=10`.

### Виджет «Воронка»
Bar chart (ступенчатый): `GET /api/analytics/funnel`.
4 столбца: сообщения → привязаны → заказы → выполнены.

### Виджет «Прогноз»
Line chart с доверительной полосой: `GET /api/analytics/forecast`.
Три линии: predicted, lower_bound, upper_bound.
Если `data = null` — скрыть виджет.

### Виджет «Retention» (отложено до 5.1)
Не реализовывать в этой версии. Скрыть из макета.
- API `GET /api/analytics/retention` не вызывать
- План на 5.1: heatmap через canvas, строки = недельные когорты, столбцы = недели после первой покупки

## 3. Селектор периода

Выпадающий список: «7 дней», «30 дней», «90 дней», «всё время».
По умолчанию: 30 дней.
При смене — все виджеты перезагружаются с новым `days`.

## 4. Обновление

- Auto-poll summary каждые 60 секунд
- Кнопка «Обновить» в шапке — ручное обновление всех данных

## 5. Состояния

| Состояние | Поведение |
|---|---|
| Loading | Skeleton для каждого виджета |
| Error | Toast + виджет с ошибкой + кнопка повтора |
| Empty | «Нет данных» |
| No analytics deps | Скрыть прогноз, остальные работают |
| Retention отложен | Виджет отсутствует в разметке |

## 6. Критерии приёмки

- Все KPI отображаются корректно
- Графики рендерятся (Chart.js)
- Смена периода перезагружает данные
- Auto-poll обновляет сумму
- Состояния loading/error/empty обработаны
- Нет падений при пустой БД
- responsive (графики адаптируются к ширине)
