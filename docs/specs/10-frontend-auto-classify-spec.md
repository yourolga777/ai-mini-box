## 2. Требования

### 2.1 Кнопка на странице сообщений (`Messages.tsx`)

Разместить **рядом со строкой поиска** (над таблицей сообщений).

- Текст: **«Распределить всё»**
- При нажатии: вызов `api.assignAllCategories(50)`
- После ответа: показать **toast-уведомление** с результатом:

  > ✅ Обработано: 45, назначено в папки: 32

- Если LLM-плагин выключен (502): показать тост-ошибку:

  > ❌ ИИ-плагин не запущен. Включите его в разделе «Плагины».

- Кнопка **disabled** во время загрузки, текст меняется на «Распределяю…»

### 2.2 Дублирующая кнопка на странице плагина (опционально)

В `PluginDetail.tsx`, в секции «Папки» (FolderSection), добавить такую же кнопку. Поведение идентичное п. 2.1.

### 2.3 Улучшить ответ эндпоинта (web-разработчик, опционально)

Сейчас `POST /api/llm/assign-all` возвращает кортеж `(processed, assigned)` как есть. Фронтенд получает `{"processed": [45, 32]}`.

**Исправить:** возвращать словарь:

```json
{
  "checked": 45,
  "assigned": 32
}
```

Соответственно поправить тип в `client.ts`.

---

## 3. Acceptance criteria

- [ ] Кнопка видна на странице сообщений
- [ ] При нажатии вызывается `/api/llm/assign-all?limit=50`
- [ ] Пока запрос идёт, кнопка disabled с текстом «Распределяю…»
- [ ] После успеха — toast с количество проверенных и назначенных
- [ ] При 502 — toast с ошибкой «ИИ-плагин не запущен»
- [ ] Эндпоинт возвращает `{ checked, assigned }`, а не `[processed, assigned]`
