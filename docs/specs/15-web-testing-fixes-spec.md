## Задача В-8: Коды ошибок в 502-ответах LLM (header)

**Где:** `packages/web/ai_mini_box_web/routers/llm_folders.py`

**Проблема:** Фронтенд не может программно определить тип ошибки — `detail` содержит только человекочитаемый текст. Клиенту (`client.ts`) и обработчикам `onError` нужен машинный код.

**Что сделать:**
Добавить `headers={"X-Error-Code": "<CODE>"}` в 3 `HTTPException(502, ...)`:

| Строка | Эндпоинт | Текущий detail | Код |
|---|---|---|---|
| 66 | `_llm_models()` | `"LLM plugin not installed"` | `ERR_LLM_NOT_INSTALLED` |
| 213 | `run_auto_processor()` | `"AutoProcessor service not available..."` | `ERR_LLM_NOT_RUNNING` |
| 296 | `assign_all()` | `"AutoProcessor service not available..."` | `ERR_LLM_NOT_RUNNING` |

Пример:
```python
# Было:
raise HTTPException(502, detail="LLM plugin not installed")
# Стало:
raise HTTPException(502, detail="LLM plugin not installed", headers={"X-Error-Code": "ERR_LLM_NOT_INSTALLED"})
```

`detail` остаётся строкой — фронтенд не ломается. Заголовки игнорируются пока, но доступны для умных обработчиков в будущем.

**Acceptance criteria:**
- Все 3 502-ответа содержат заголовок `X-Error-Code` с корректным кодом
- Формат `detail` не изменился (строка)
- Существующие обработчики ошибок на фронтенде не сломаны

## Задача В-9: Healthcheck `GET /api/llm/health`

**Где:** `packages/web/ai_mini_box_web/routers/llm_folders.py`

**Что сделать:**
Добавить новый эндпоинт `GET /api/llm/health` (~25 строк кода).

Логика:
1. Попробовать `get_service("llm")`:
   - Если `None` → `{"status": "not_initialized", "model": null}`
   - Если есть → попробовать `provider.model_name` (для remote) или `provider.config.model_path` (для local):
     - Получилось → `{"status": "ok", "model": "<имя_модели>"}`
     - Не получилось → `{"status": "no_model", "model": null}`
2. `services.auto_processor`: `"running"` если `get_service("auto_processor") is not None`, иначе `"stopped"`

Формат ответа:
```json
{
  "status": "ok" | "not_initialized" | "no_model",
  "model": "gpt-4o" | "llama-2-7b.gguf" | null,
  "services": {
    "auto_processor": "running" | "stopped"
  }
}
```

**Acceptance criteria:**
- `GET /api/llm/health` возвращает HTTP 200
- При работающем LLM: статус `ok`, модель определена, `auto_processor: "running"`
- При остановленном LLM: статус `not_initialized`
- При ошибке загрузки модели: статус `no_model`
