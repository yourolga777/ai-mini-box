# Спецификация: Версия плагина и обновление каталога

**Для:** frontend-разработчик + web-разработчик  
**Приоритет:** P4  
**Затрагиваемые плагины:** все плагины (общая страница «Плагины»)

---

## 1. Проблема

В справке (`08-plugins.md`) указано:

- **«Версия — установленная и последняя доступная»** — сейчас для установленных плагинов показывается версия из каталога (последняя доступная), а не реальная установленная
- **«Нажмите «Установить» — программа скачает и подключит его»** — нет кнопки ручного обновления каталога

---

## 2. Требования

### 2.1 Отображение версий (раздельно)

В карточке установленного плагина показывать две строки:

```
v2.1.0 (установлена) → v2.3.0 (доступна)
```

Если версии совпадают — показывать только одну: `v2.1.0`

**Что нужно сделать:**

- Web-бэкенд: в ответе `GET /api/plugins` добавить поле `installed_version` (реальная версия из `importlib.metadata.version(package)`) отдельно от `version` (последняя из каталога)
- Фронтенд: отображать обе версии, подсвечивать `has_update` если они различаются

### 2.2 Кнопка «Обновить каталог»

Добавить кнопку **«Обновить каталог»** (или иконку ↻) на вкладке «Доступные» в `Plugins.tsx`.

- При нажатии: инвалидировать кеш `useQuery` для ключа `["catalog"]`
- Пока загружается: кнопка disabled, текст «Загружаю…»
- После загрузки: список обновится автоматически

### 2.3 Бэкенд (опционально)

Если `GET /api/plugins/catalog` кеширует ответ на бэкенде — добавить `Cache-Control: no-cache` когда передан параметр `?refresh=1`, либо сделать `POST /api/plugins/catalog/refresh`.

---

## 3. Acceptance criteria

- [ ] В карточке установленного плагина видны две версии (установленная и доступная)
- [ ] Если есть обновление — это визуально заметно и кнопка «Обновить» активна
- [ ] На вкладке «Доступные» есть кнопка ручного обновления каталога
- [ ] После обновления каталога список актуализируется
