# Спецификация: Фильтр заказов по контакту

**Для:** frontend-разработчик  
**Приоритет:** P4  
**Затрагиваемые плагины:** не затрагивает — изменение в общем разделе «Заказы»

---

## 1. Проблема

В справке (`04-orders.md:13`) указано: «Фильтровать — по статусу, по контакту, по тексту заметок». Фильтр по статусу и поиск по заметкам реализованы, а фильтр/выбор по контакту — нет. Параметр `contact_id` в API `/api/orders` существует, но в UI (`Orders.tsx`) нет контрола для выбора контакта.

---

## 2. Требования

### 2.1 Добавить выпадающий список контактов в фильтры

В `Orders.tsx`, в строке фильтров (рядом с селектом статуса и поиском по заметкам), добавить `<select>` или `<Combobox>` для выбора контакта.

- По умолчанию: **«Все контакты»**
- Список контактов загружается из `GET /api/contacts` (уже есть в компоненте как `contactMap`)
- При выборе контакта: отправлять `?contact_id=X` в API
- При смене контакта: сбрасывать пагинацию на первую страницу

### 2.2 Бэкенд

Параметр `contact_id` уже объявлен в `orders.py:64`. Проверить, что фильтрация работает корректно (`ContactFilter.contact_id`).

---

## 3. Acceptance criteria

- [ ] В строке фильтров заказов появился выпадающий список контактов
- [ ] По умолчанию выбран «Все контакты»
- [ ] При выборе контакта показываются только его заказы
- [ ] Работает вместе с фильтром по статусу и поиском по заметкам
