# Спецификация: Telegram-плагин — Рекомендации после тестирования (v1)

**Разработчик:** Telegram-плагин  
**Приоритет:** P2  
**Статус:** Рекомендации (не баги)

---

## Результат анализа

Критических багов в Telegram-плагине не обнаружено. Плагин корректно:
- Принимает токен через UI (PluginDetail.tsx)
- Запускает/останавливает демон
- Показывает список разрешённых чатов
- Позволяет делать опрос (poll)

---

## Рекомендация: Автоматический poll новых сообщений

**Проблема:** После настройки плагина пользователь должен вручную нажимать «Опрос», чтобы получить сообщения. В production-режиме ожидается постоянный polling.

**Что сделать:**
- Добавить в `telegram_daemon.py` фоновый polling с интервалом (например, 10-30 сек) когда демон запущен
- При получении нового сообщения — сохранять его в `MessageModel` и триггерить `auto_processor` (если доступен)
- Кнопку «Опрос» оставить как ручной триггер

**Acceptance criteria:**
- При запущенном демоне сообщения приходят автоматически
- Интервал polling настраивается в конфиге плагина
- Кнопка «Опрос» по-прежнему работает как ручной вызов

---

## Границы ответственности

- Не менять фронтенд (PluginDetail.tsx уже умеет показывать статус демона)
- Не менять другие плагины
- Изменения опциональны (но желательны для production)
