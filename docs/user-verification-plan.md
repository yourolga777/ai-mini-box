# План верификации: AI Box — проверка работоспособности

После перезапуска сервера выполните шаги ниже, чтобы убедиться,
что приложение и все плагины работают корректно.

---

## 1. Базовая проверка веб-интерфейса

- [ ] Откройте `http://127.0.0.1:8080` в браузере
- [ ] Панель (Dashboard) загружается — видны 4 карточки KPI (сообщения, контакты, заказы, выручка)
- [ ] Клик на каждую секцию бокового меню — страницы открываются без ошибок
- [ ] **Справка** — открыть, проверить, что разделы отображаются

## 2. Сообщения

- [ ] **Список сообщений** — открыть `/messages`
- [ ] Видна колонка **"Категория"** с бейджами (синий ЗАКАЗ, зелёный ВОПРОС и т.д.)
- [ ] Фильтр **"Все категории"** работает — выберите "ЗАКАЗ", таблица отфильтруется
- [ ] Бейджи **🤖** и **👤** отображаются (если есть автоответы или need_human)
- [ ] **Карточка сообщения** — клик по любому сообщению
- [ ] Если есть автоответ — виден зелёный блок "🤖 Автоматический ответ отправлен"
- [ ] Если есть need_human — виден красный блок "👤 Требует внимания оператора"
- [ ] Видна кнопка **"🔄 Переобработать через ИИ"** (работает только при запущенном LLM-плагине)
- [ ] **Поиск** — введите текст в строку поиска, таблица фильтруется
- [ ] **Привязать к контакту** — нажмите "Привязать", найдите контакт, выберите

## 3. Настройки бизнеса

- [ ] **Страница настроек** — кликнуть "Бизнес" в боковом меню
- [ ] Форма загружается с текущими значениями
- [ ] Измените поле (например, "Часы работы"), нажмите **"Сохранить"**
- [ ] Перезагрузите страницу — изменения сохранились
- [ ] **FAQ-редактор** — нажмите "+ Добавить вопрос"
- [ ] Заполните вопрос и ответ, нажмите "Сохранить"
- [ ] Удалите запись FAQ кнопкой **×**, сохраните

## 4. Контакты

- [ ] **Список контактов** — открыть `/contacts`
- [ ] Клик по контакту — открывается карточка
- [ ] В карточке видны сообщения контакта и заказы
- [ ] Чат-диалог отображает историю сообщений (входящие — слева, исходящие — справа)

## 5. Заказы

- [ ] **Список заказов** — открыть `/orders`
- [ ] Фильтр по статусу работает
- [ ] Клик по заказу — открывается страница заказа
- [ ] Видны позиции заказа (OrderItems), контакт, источник
- [ ] Кнопка **"Создать заказ из сообщения"** в MessageDetail — создаёт заказ

## 6. Плагины

- [ ] **Раздел "Плагины"** — открыть `/plugins`
- [ ] Видны вкладки **"Установленные"** и **"Доступные"**
- [ ] Для каждого плагина отображается версия, статус, описание
- [ ] Telegram-бот: нажмите **"Проверить сейчас"** — сообщения подгружаются
- [ ] Telegram-бот: нажмите **"Настроить"** — открывается форма конфигурации
- [ ] LLM-плагин: нажмите **"Настроить"** — видны поля провайдера, модели
- [ ] Кнопка **"Распределить всё"** — запускает авто-классификацию (требует LLM)
- [ ] Если плагин не запущен — кнопки "Запустить" и "Остановить" работают

## 7. Бизнес-процесс: Сообщение → Автоответ → Заказ

- [ ] Отправьте сообщение в Telegram-бот (если настроен)
- [ ] В AI Box сообщение появляется в разделе "Сообщения"
- [ ] Если LLM-плагин запущен — сообщение получает категорию
- [ ] Если ИИ решил ответить — в Telegram приходит автоответ
- [ ] Если в сообщении был заказ — в разделе "Заказы" создаётся новый заказ
- [ ] В MessageDetail виден зелёный блок автоответа и заказ

## 8. Edge-кейсы

- [ ] **LLM-плагин выключен**: сообщение сохраняется, категория не назначается
- [ ] **Неизвестный чат**: сообщение из незарегистрированного чата игнорируется
- [ ] **Пустые настройки бизнеса**: страница BusinessSettings загружается
- [ ] **404**: открыть `/messages/99999` — "Сообщение не найдено"
- [ ] **Сортировка по need_human**: `GET /api/messages?sort=need_human:desc` — сначала требующие оператора

## Быстрая проверка через API (curl / httpie)

```bash
# Health
curl http://127.0.0.1:8080/api/health

# Business config
curl http://127.0.0.1:8080/api/business/config

# Messages with need_human sort
curl "http://127.0.0.1:8080/api/messages?sort=need_human:desc&limit=5"

# Messages with category filter
curl "http://127.0.0.1:8080/api/messages?category=ЗАКАЗ"

# Reprosess message (replace 1 with actual message ID)
curl -X POST http://127.0.0.1:8080/api/messages/1/reprocess-chatbot
```
